## 架构对比

### Go 版本特点
- **语言**: Go 1.22+
- **并发**: Goroutines + Channels
- **HTTP 框架**: net/http
- **依赖**: Go modules

### Rust 版本特点
- **语言**: Rust (安全、并发)
- **异步**: Tokio (异步运行时)
- **HTTP 框架**: Axum (类型安全路由)
- **内存安全**: 编译时保证，无 GC

## 性能优势 (Rust)

1. **内存效率**: 无 GC，内存占用更低
2. **启动速度**: 编译为原生代码，启动更快
3. **并发安全**: 编译时保证线程安全
4. **类型安全**: 编译时捕获更多错误

## 部署建议

1. **Docker 镜像**: 使用多阶段构建优化镜像大小
2. **资源限制**: 建议至少 512MB 内存
3. **CPU**: 单核即可，多核可提升并发性能
4. **持久化**: memfs 数据在内存中，重启丢失
